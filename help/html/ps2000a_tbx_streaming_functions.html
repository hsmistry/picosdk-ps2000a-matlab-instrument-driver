autoStopped

autoStop = autoStopped(obj)

autoStopped(obj) indicates if the device has stopped on collection of the number of samples specified in the call to the ps2000aRunStreaming function.

The autoStop property must have been set to 1.

Input Arguments:

  • obj - the object representing the Streaming group.

Output Arguments:

  • autoStop - 0 if the device has not stopped, non-zero if streaming has auto-stopped.

Example:

    hasAutoStopped = invoke(streamingGroupObj, 'autoStopped');

See also: ps2000aRunStreaming

availableData

[numSamples, startIndex] = availableData(obj)

availableData(obj) indicates the number of samples returned from the driver and shows the start index of the data in the buffer when collecting data in streaming mode.

Input Arguments:

  • obj - the object representing the Streaming group.

Output Arguments:

  • numSamples - 0, data is not yet available; Non-zero, the number of samples returned from the driver.
  • startIndex - an index to the first valid sample in the buffer (when data is available). The startIndex value is zero-based so should be incremented by 1 when used to retrieve data from the buffer.

Example:

    [numSamples, startIndex] = invoke(streamingGroupObj, 'availableData');

clearTriggerReady

triggerCleared = clearTriggerReady(obj)

clearTriggerReady(obj) clears the ps2000aWrap shared library's triggered and triggeredAt flags in relation to streaming mode capture.

Input Arguments:

  • obj - the object representing the Streaming group.

Output Arguments:

  • 1 - Function successfully completed.

Example:

    triggerCleared = invoke(streamingGroupObj, 'clearTriggerReady');

See also: isTriggerReady

getStreamingLatestValues

status = getStreamingLatestValues(obj)

getStreamingLatestValues(obj) facilitates communication with the underlying driver to return the next block of values to your application when capturing data in streaming mode.

Input Arguments:

  • obj - the object representing the Streaming group.

Output Arguments:

  • status - status code returned by the driver.

Note: This function should be called in a loop with the isReady function as shown in the PS2000A_ID_Streaming_Example.m script.

Example:

    status = invoke(streamingGroupObj, 'getStreamingLatestValues');

See also: isReady

isReady

ready = isReady(obj)

isReady(obj) is used to poll the driver to verify that data is ready to be received. The ps2000aRunStreaming function must have been called prior to calling this function.

Input Arguments:

  • obj - the object representing the Streaming group.

Output Arguments:

  • ready - 0, if data is not yet available; non-zero if data is ready to be collected.

Note: This function should be called in a loop with the getStreamingLatestValues function as shown in the PS2000A_ID_Streaming_Example.m script.

Example:

    ready = invoke(streamingGroupObj, 'isReady');

See also: getStreamingLatestValues

isTriggerReady

[triggered, triggeredAt] = isTriggerReady(obj)

isTriggerReady(obj) indicates whether a trigger has occurred when collecting data in streaming mode, and the location of the trigger point.

Input Arguments:

  • obj - the object representing the Streaming group.

Output Arguments:

  • triggered - 0, if the device has not triggered; Non-zero, if the device has been triggered.
  • triggeredAt - the zero-based index of the sample in the buffer relative to the start index (see availableData) where the trigger occurred.

Example:

    [triggered, triggeredAt] = invoke(streamingGroupObj, 'isTriggerReady');

See also: availableData | setSimpleTrigger

ps2000aNoOfStreamingValues

[status, noOfValues] = ps2000aNoOfStreamingValues(obj)

ps2000aNoOfStreamingValues(obj) returns the number of samples available after data collection in streaming mode. Call this function after calling ps2000aStop.

Input Arguments:

  • obj - the object representing the Streaming group.

Output Arguments:

  • status - status code returned by the driver.
  • noOfValues - the number of samples.

Example:

    [status, noOfValues] = invoke(ps2000aDeviceObj, 'ps2000aNoOfStreamingValues');

See also: ps2000aStop

ps2000aRunStreaming

[status, sampleInterval, sampleIntervalTimeUnitsStr] = ps2000aRunStreaming(obj, downSampleRatio, downSampleRatioMode, bufferLength)

ps2000aRunStreaming(obj, downSampleRatio, downSampleRatioMode, overviewBufferSize) tells the oscilloscope to start collecting data in streaming mode. Use the function getStreamingLatestValues to ensure values are copied into the application buffers.

The numPreTriggerSamples and numPostTriggerSamples Device Properties, as well as the autoStop and streamingInterval properties should be set prior to calling this function, otherwise the current values will be used.

Regardless of whether a trigger is set or not set, the maximum number of samples held by the underlying ps2000a shared library will be the sum of numPreTriggerSamples and numPostTriggerSamples.

Input Arguments:

  • obj - the object representing the Streaming group.
  • downSampleRatio - the downsampling factor that will be applied to the raw data.
  • downSampleRatioMode - which downsampling mode to use (an enumeration from the enPS2000ARatioMode range).
  • overviewBufferSize - the size of the overview buffers (the buffers passed by the application to the shared library). The size must be less than or equal to the bufferLth value passed to ps2000aSetDataBuffer).

Output Arguments:

  • status - status code returned by the driver.
  • sampleInterval - the actual time interval used.
  • sampleIntervalTimeUnitsStr - a string representing the time unit for the sampling interval.

Example:

    % Set Streaming group object properties
    set(ps2000aDeviceObj, 'numPreTriggerSamples', 0);
    set(ps2000aDeviceObj, 'numPostTriggerSamples', 10e6);

    set(streamingGroupObj, 'streamingInterval', 1e-6);  % 1 microsecond (1 MS/s)
    set(streamingGroupObj, 'autoStop', 1);              % Stop after collecting the desired number of samples

    % Set parameters and call the function
    downSampleRatio     = 1;
    downSampleRatioMode = ps2000aEnuminfo.enPS2000ARatioMode.PS2000A_RATIO_MODE_NONE;
    bufferLength        = 250000;

    [status, sampleInterval, sampleIntervalTimeUnitsStr] = invoke(ps2000aDeviceObj, 'ps2000aRunStreaming', downSampleRatio, ...
                                                                downSampleRatioMode, bufferLength)

See also: getStreamingLatestValues

setAppAndDriverBuffers

status = setAppAndDriverBuffers(obj, channel, appBuffer, driverBuffer, bufferLength)

setAppAndDriverBuffers(obj, channel, appBuffer, driverBuffer, bufferLength) set the application and corresponding driver buffer in the wrapper shared library for an analog input channel.

This function is required to allow the driver to copy the data into the application buffer being used in MATLAB when acquiring data from the PicoScope in streaming mode without aggregation.

Input Arguments:

  • obj - the object representing the Streaming group.
  • channel - the analog channel to which the buffer corresponds (an enumeration from the enPS2000AChannel range).
  • appBuffer - a libpointer object corresponding to a 1-D array of 16-bit integer values representing the application buffer.
  • driverBuffer - the libpointer object passed to the driver in the ps2000aSetDataBuffer function call for this channel.
  • bufferLength - the length of the arrays represented by the libpointer objects (must be equal).

Output Arguments:

  • status - status code returned by the wrapper driver.

Example:

    % Create a driver buffer for channel A and register it with the driver
    channelA            = ps2000aEnuminfo.enPS2000AChannel.PS2000A_CHANNEL_A;
    bufferLength        = 250000; % Size of the buffer to collect data from the driver.
    pDriverBufferChA    = libpointer('int16Ptr', zeros(bufferLength, 1, 'int16'));
    segmentIndex        = 0;
    ratioMode           = ps2000aEnuminfo.enPS2000ARatioMode.PS2000A_RATIO_MODE_NONE;

    status = invoke(ps2000aDeviceObj, 'ps2000aSetDataBuffer', channelA, pDriverBufferChA, bufferLength, ...
                segmentIndex, ratioMode);

    % Create an application buffer for channel A
    pAppBufferChA = libpointer('int16Ptr', zeros(bufferLength, 1, 'int16'));

    % Register the buffers with the wrapper driver
    status = invoke(streamingGroupObj, 'setAppAndDriverBuffers', channelA, pAppBufferChA, pDriverBufferChA, bufferLength);

See also: ps2000aSetDataBuffer | setMaxMinAppAndDriverBuffers

setAppAndDriverDigiBuffers

status = setAppAndDriverDigiBuffers(obj, port, appDigiBuffer, driverDigiBuffer, digiBufferLength)

setAppAndDriverDigiBuffers(obj, port, appBuffer, driverBuffer, bufferLength) set the application and corresponding driver buffer for a digital port in the wrapper driver.

This function is required to allow the driver to copy the data for a digital port into the corresponding application buffer being used in MATLAB when acquiring data from the PicoScope in streaming mode without aggregation.

Input Arguments:

  • obj - the object representing the Streaming group.
  • port - the port to which the buffer corresponds (an enumeration from the enPS2000AWrapDigitalPortIndex range).
  • appDigiBuffer - a libpointer object corresponding to a 1-D array of 16-bit integer values representing the application buffer.
  • driverDigiBuffer - the libpointer object passed to the driver in the ps2000aSetDataBuffer function call for this port.
  • bufferLength - the length of the arrays represented by the libpointer objects (must be equal).

Output Arguments:

  • status - status code returned by the wrapper driver.

Example:

    % Create a driver buffer for port DPORT0 and register it with the driver
    dPort0              = ps2000aEnuminfo.enPS2000ADigitalPort.PS2000A_DIGITAL_PORT0;
    digiBufferLength    = 100000; % Size of the buffer to collect data from the driver.
    segmentIndex        = 0;
    ratioMode           = ps2000aEnuminfo.enPS2000ARatioMode.PS2000A_RATIO_MODE_NONE;

    pDriverBufferDPort0 = libpointer('int16Ptr', zeros(digiBufferLength, 1, 'int16'));

    status = invoke(ps2000aDeviceObj, 'ps2000aSetDataBuffer', dPort0, pDriverBufferDPort0, ...
        digiBufferLength, segmentIndex, ratioMode);

    % Create an application buffer for channel A
    pAppBufferDPort0 = libpointer('int16Ptr', zeros(digiBufferLength, 1, 'int16'));

    % The wrapper uses a zero-based enumeration to identify the digital port
    wrapperDPort0 = ps2000aWrapEnuminfo.enPS2000AWrapDigitalPortIndex.PS2000A_WRAP_DIGITAL_PORT0;

    % Register the buffers with the wrapper driver
    status = invoke(streamingGroupObj, 'setAppAndDriverDigiBuffers', wrapperDPort0, ...
        pAppBufferDPort0, pDriverBufferDPort0, digiBufferLength);

See also: ps2000aSetDataBuffer | setMaxMinAppAndDriverDigiBuffers

setMaxMinAppAndDriverBuffers

status = setMaxMinAppAndDriverBuffers(obj, channel, appMaxBuffer, appMinBuffer, ...
            driverMaxBuffer, driverMinBuffer, bufferLength)

setMaxMinAppAndDriverBuffers(obj, channel, appMaxBuffer, appMinBuffer, driverMaxBuffer, driverMinBuffer, bufferLength) set the maximum and minimum application and corresponding driver buffers for an analog input channel in the wrapper driver.

This function is required to allow the driver to copy the data into the maximum and minimum application buffers being used in MATLAB when acquiring data from the PicoScope in streaming mode with aggregation.

Input Arguments:

  • obj - the object representing the Streaming group.
  • channel - the analog channel to which the buffers correspond (an enumeration from the enPS2000AChannel range).
  • appMaxBuffer - a libpointer object corresponding to a 1-D array of 16-bit integer values to hold the maximum aggregated data values.
  • appMinBuffer - a libpointer object corresponding to a 1-D array of 16-bit integer values to hold the minimum aggregated data values.
  • driverMaxBuffer - the libpointer object passed to the driver for the maximum aggregated data values in the ps2000aSetDataBuffers function call.
  • driverMinBuffer - the libpointer object passed to the driver for the minimum aggregated data values in the ps2000aSetDataBuffers function call for this channel.
  • bufferLength - the length of the arrays represented by the libpointer objects (must be equal).

Output Arguments:

  • status - status code returned by the wrapper driver.

Example:

    % Create driver buffers for channel A and register them with the driver
    channelA            = ps2000aEnuminfo.enPS2000AChannel.PS2000A_CHANNEL_A;
    bufferLength        = 250000; % Size of the buffer to collect data from the driver.
    pDriverBufferChAMax = libpointer('int16Ptr', zeros(digiBufferLength, 1, 'int16'));
    pDriverBufferChAMin = libpointer('int16Ptr', zeros(digiBufferLength, 1, 'int16'));
    segmentIndex        = 0;
    ratioMode           = ps2000aEnuminfo.enPS2000ARatioMode.PS2000A_RATIO_MODE_NONE;

    status = invoke(ps2000aDeviceObj, 'ps2000aSetDataBuffers', channelA, pDriverBufferChAMax, ...
                pDriverBufferChAMin, bufferLength, segmentIndex, ratioMode);

    % Create max./min. application buffers for channel A
    pAppBufferChAMax = libpointer('int16Ptr', zeros(bufferLength, 1, 'int16'));
    pAppBufferChAMin = libpointer('int16Ptr', zeros(bufferLength, 1, 'int16'));

    % Register the buffers with the wrapper driver
    status = invoke(streamingGroupObj, 'setAppAndDriverBuffers', channelA, pAppBufferChAMax, pAppBufferChAMin, ...
                pDriverBufferChAMax, pDriverBufferChAMin, bufferLength);

See also: ps2000aSetDataBuffers | setAppAndDriverBuffers
