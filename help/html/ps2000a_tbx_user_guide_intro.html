PicoScope 2000 Series (A API) - MATLAB Generic Instrument Driver User Guide

The PicoScope 2000 Series (A API) - MATLAB Generic Instrument Driver provides functionality to configure a PicoScope 2000 Series oscilloscope or mixed signal oscilloscope through various properties and functions. This allows you to collect data using various sampling modes as well as providing the ability to configure and operate the signal generator.

General Procedure

A typical Instrument session for data acquisition will consist of the following steps:

  • Open a connection to the device.
  • Configure the device (channels, triggering, and if required, the signal generator).
  • Capture data.
  • Process data.
  • Close the connection to the device.

It is not necessary to configure the channels and triggering if only the signal generator feature is required instead of data acquisition.

Voltage Ranges

Analog input channels

You can set a device input channel to any voltage range from ±20 mV to ±20 V (subject to the device specification) with ps2000aSetChannel. If you are not using the functions provided by the Instrument Driver that return the data values converted to millivolts, each sample is scaled to 16 bits, and the minimum and maximum values returned to your application are given by the Instrument Driver's minADCValue and maxADCValue properties as follows:

Property Voltage Value returned
decimal hex
maxADCValue maximum 32512 7F00
zero 0 0000
minADCValue minimum -32512 8100

Note: Oscilloscope probe and current clamp settings are not taken into account, so the values returned from the functions will need to be scaled accordingly.

External trigger input (PicoScope 2206, 2207 and 2208 only)

The external trigger input (marked EXT), has a ±5 V input.

MSO Digital Data

This section applies to mixed-signal oscilloscopes (MSOs) only.

A PicoScope MSO has two 8-bit digital ports—PORT0 and PORT1—making a total of 16 digital channels. The data from each port is returned in a separate buffer that is set up by the ps2000aSetDataBuffer and ps2000aSetDataBuffers functions. For compatibility with the analog channels, each buffer is an array of 16-bit words. The 8-bit port data occupies the lower 8 bits of the word, and the upper 8 bits of the word are undefined.

Triggering

PicoScope oscilloscopes can either start collecting data immediately or be programmed to wait for a trigger event - use the setTriggerOff or setSimpleTrigger functions respectively from the Trigger group in order to do this.

A trigger event can occur when one of the signal or trigger input channels crosses a threshold voltage on either a rising or a falling edge.

Note: Advanced trigger functionality is not currently supported by the Instrument Driver.

Downsampling Modes

Various methods of data reduction, or downsampling, are possible with the PicoScope 2000 Series oscilloscopes. The downsampling is done at high speed by dedicated hardware inside the scope, making your application faster and more responsive than if you had to do all the data processing in software.

You specify the downsampling mode when you call one of the data collection functions such as ps2000aGetValues. The following modes are available:

Ratio Mode Enumeration Value Description
PS2000A_RATIO_MODE_NONE 0 No downsampling is performed. The downSampleRatio parameter in the relevant function call is ignored.
PS2000A_RATIO_MODE_AGGREGATE 1 Reduces every block of n values to just two values: a minimum and a maximum. The minimum and maximum values are returned in two separate buffers.
PS2000A_RATIO_MODE_DECIMATE 2 Reduces every block of n values to just the first value in the block, discarding all the other values.
PS2000A_RATIO_MODE_AVERAGE 4 Reduces every block of n values to a single value representing the average (arithmetic mean) of all the values. Equivalent to the 'oversampling' function on older scopes.
